JWT Authentication System
Admin and student login, JWT signing, httpOnly cookies, middleware


■ OBJECTIVE
Build the complete authentication layer. Admin and student have separate login flows. JWTs are stored in httpOnly
cookies (not localStorage) for XSS protection. Middleware validates JWT on protected routes. bcrypt hashes all
passwords.

■ FILES TO CREATE / EDIT
   • backend/src/routes/auth.routes.ts
   • backend/src/controllers/auth.controller.ts
   • backend/src/middleware/authMiddleware.ts
   • backend/src/middleware/roleMiddleware.ts
   • frontend/src/lib/apiClient.ts
   • frontend/src/hooks/useAuth.ts


■ GITHUB COPILOT PROMPT

  Build the complete JWT authentication system for the EdTech platform. Backend: 1. POST /api/auth/admin/login —
  fetch admin by email from Supabase, bcrypt.compare password, sign JWT {id, role:'admin', email} with
  JWT_SECRET (7 day expiry), set httpOnly cookie 'token'. 2. POST /api/auth/student/login — same flow for
  students table, role:'student'. 3. POST /api/auth/logout — clear the httpOnly cookie. 4. GET /api/auth/me — verify
  JWT from cookie, return user object. 5. authMiddleware.ts — extract JWT from cookie, verify, attach req.user = {id,
  role, email}. Return 401 if missing/invalid. 6. roleMiddleware.ts — factory function: requireRole('admin') returns
  middleware that checks req.user.role, returns 403 if wrong role. 7. Rate limiter on login routes: max 10 attempts per
  IP per 15 minutes using express-rate-limit + Redis store. Frontend: 8. frontend/src/lib/apiClient.ts — axios instance
  with baseURL=NEXT_PUBLIC_API_URL, withCredentials:true (to send cookies), interceptor that redirects to /login
  on 401. 9. frontend/src/hooks/useAuth.ts — React hook with login(email, password, role), logout(), user state,
  isLoading. Calls /api/auth/me on mount to restore session. Install required packages: bcryptjs, jsonwebtoken,
  express-rate-limit, cookie-parser, axios. TypeScript types for all functions. No any types.


■ KEY TASKS COPILOT WILL COMPLETE
   ✓ Admin and student login endpoints with bcrypt verification
   ✓ JWT signing with httpOnly cookie storage
   ✓ authMiddleware and roleMiddleware for route protection
   ✓ Rate limiting on login (10 attempts / 15 min)
   ✓ Frontend apiClient with credential forwarding and 401 redirect
   ✓ useAuth hook with session restoration on mount


  ■ SUCCESS CRITERIA
